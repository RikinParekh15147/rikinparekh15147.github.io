Software Defined Networking

SDN

Network behaviour controlled via software, not hardware config
Separates Control Plane (what to do) from Data Plane (doing it)
Enables automation, rapid changes, cloud-scale networking
Concept, not a product

Cloud Platforms

OpenStack & Kubernetes

☁️ OpenStack IaaS
Full private cloud platform
Manages VMs, block storage, object storage, networking
Uses OVN/OVS via Neutron networking service
Deploys on bare-metal in data centres
→ Tenant = VPC-like isolated network
⚙️ Kubernetes CaaS
Container orchestration platform
Manages deployment, scaling, and networking of containers
Uses CNI plugins (Calico, Cilium, OVN-Kubernetes)
Each Pod gets an IP inside a virtual network
→ Namespace = isolated network scope
Both rely on SDN / OVN / OVS concepts to deliver virtual networking

Security

Best Practices

🔒
Keep DB Private
RDS should never sit in a public subnet. No direct internet route. Only EC2/app servers may reach it.
🚫
No Open Access
Never allow 0.0.0.0/0 on sensitive ports (22, 3306, 5432). Use specific CIDR ranges or SG references.
🗓️
Multi-AZ Redundancy
Spread subnets across ≥ 2 AZs. Enables failover — any single AZ failure won't take you down.
🧱
Restrict Ports
Allow only the minimum ports needed. Web: 443. App: internal only. DB: port from app tier only.
